# 译 · Chinese Translator

A local, ChatGPT-style web app: paste a chat log (Chinese / English / emojis / mixed,
multiple speakers) and get each line back with **pinyin**, **English translation**, and
**context notes** for idioms and slang — streamed live over a WebSocket to every
connected device (phone, PC, Mac) at once.

- **Translation**: your local LLM via LM Studio's OpenAI-compatible API.
- **Pinyin**: `pypinyin` (deterministic tone marks — not the LLM).
- **Server**: FastAPI + uvicorn (ASGI), async streaming with `httpx`.
- **Shared session**: one synchronized feed across all devices; `>`-prefixed
  messages are questions to the assistant instead of text to translate.
- **Model selector** + **click-to-speak** (neural TTS) in the UI.
- **Browser extension** (`extension/`): hover any Chinese on any web page for the
  same pinyin + translation + notes in a popup.

## Requirements

1. **LM Studio** running with a chat model loaded (e.g. `qwen/qwen3.6-35b-a3b`).
   - Tip: disable idle auto-unload so the model stays resident (kills first-token lag).
2. LM Studio's local server started (**Developer → Start Server**, port 1234).

## Quick start (Makefile)

```bash
make install     # create venv + install deps (first time only)
make dev         # run with auto-reload
```

Then open **http://MLTX-TWANG.local:5001** (or `http://localhost:5001` on the Mac).

| Target | Does |
|--------|------|
| `make install` | create `venv/` (Python 3.14) and install `requirements.txt` |
| `make install-ml` | create `ml-venv/` (Python 3.12 via uv) and install the TTS stack |
| `make dev` | run the main app with auto-reload (uvicorn `--reload`) |
| `make run` | run the main app without auto-reload |
| `make tts` | run the Kokoro neural-TTS sidecar (127.0.0.1:5060) |
| `make stop` / `make stop-tts` | kill the app / the TTS sidecar |
| `make restart` | `stop` then `dev` |
| `make ngrok` | expose the port via ngrok (`NGROK_AUTH=user:pass` to password-protect) |

Override vars on the command line, e.g. `make dev PORT=5002` or `make run HOST=127.0.0.1`.

## Voice (click-to-speak)

Each translated Chinese line has a 🔊 — click it to hear it. Audio is generated by a
local **Kokoro** neural-TTS sidecar (`tts_service.py`) running on `ml-venv` (Python 3.12,
since the ML stack won't build on 3.14). The main app proxies `/tts` to it, so the browser
only talks to one origin (works through ngrok/Tailscale). If the sidecar isn't running, the
app falls back to the browser's built-in voice automatically.

Two-process workflow:

```bash
make install-ml   # first time only (uv + Python 3.12 + Kokoro)
make tts          # terminal 1: the voice sidecar
make dev          # terminal 2: the app
```

Default voice is `zf_xiaoxiao` (override with `TTS_VOICE=zm_yunxi make tts`, and the
`TTS_VOICE` constant in `index.html`). This same `ml-venv` will host Whisper for dictation.

## Manual run (no make)

```bash
DEV=1 ./venv/bin/python server.py                          # auto-reload
./venv/bin/uvicorn server:app --reload --host 0.0.0.0 --port 5001   # same thing
./venv/bin/python server.py                                # no reload
```

## Config (environment variables)

| Var | Default | Notes |
|-----|---------|-------|
| `HOST` | `0.0.0.0` | reachable from PC/phone on the LAN; `127.0.0.1` locks it to this Mac |
| `PORT` | `5001` | web app port |
| `DEV` | (off) | `DEV=1` enables uvicorn auto-reload |
| `LLM_BASE` | `http://localhost:1234/v1` | LM Studio's OpenAI-compatible endpoint |
| `LLM_MODEL` | `qwen/qwen3.6-35b-a3b` | fallback if model auto-detect fails |

The chat model is auto-detected from `/v1/models` (first non-embedding model), so
switching models in LM Studio needs no code change.

## Exposing it remotely

- **Your own devices from anywhere**: Tailscale (private, no public exposure).
- **A quick public link**: `make ngrok NGROK_AUTH=user:pass`. The app itself has
  **no auth** and broadcasts to all clients, so never expose it publicly without a
  password gate.

## Browser extension

`extension/` is a Manifest V3 extension: hover Chinese text on any page and a popup
shows pinyin, translation, notes, and a 🔊 — powered by the same server (it calls
`POST /translate` and `/tts`). Results are cached locally.

Load it (Chrome/Edge): `chrome://extensions` → enable Developer mode → **Load unpacked**
→ select `extension/`. Then open the extension popup and set the server URL
(default `http://MLTX-TWANG.local:5001`).

Repackage for the Web Store (publish **Unlisted** — it only works with your own server):

```bash
cd extension && zip -r ../chinese-hover-translator.zip . -x '*.DS_Store'
```

See `extension/PRIVACY.md`. Note: translation works on HTTP and HTTPS pages, but the
🔊 is blocked on HTTPS pages (mixed content) until the server is served over HTTPS
(e.g. via Tailscale).

## Files

- `server.py` — FastAPI/uvicorn server: WebSocket shared session, async LLM streaming, JSON segment extraction, pinyin; HTTP `POST /translate` and `/tts` (proxied to the sidecar).
- `index.html` — single-page web UI (feed, composer, model selector, click-to-speak).
- `tts_service.py` — Kokoro neural-TTS sidecar (runs on `ml-venv`, Python 3.12).
- `extension/` — Manifest V3 browser extension (hover-to-translate).
- `Makefile`, `requirements.txt` (main app), `requirements-ml.txt` (sidecar).
